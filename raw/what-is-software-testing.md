# What Is Software Testing, Really?

## Common Definitions

> "Software testing is a way to assess the quality of the software and to reduce the risk of software failure in operation."

> "The act of checking whether software satisfies expectations."

## Testing Is Not a Step — It's Continuous

Testing spans entire SDLC:
- Requirements analysis
- System design
- Coding
- Testing
- Deployment
- Monitoring in production

**Key insight:** Testing against different artifacts throughout the cycle, not just at the end.

## V-Model Insight

Verification and validation happen **repeatedly across the whole cycle**:
- Static testing (reviews, analysis) = just as important as dynamic
- Challenge requirements and design = testing
- Debugging and log analysis = testing

## Testing vs QA

| QA (Quality Assurance) | Testing |
|----------------------|---------|
| Process improvement | Specific software evaluation |
| Prevents defects | Finds defects |
| How to build quality software | Is this particular build quality? |

**QA activities:** Define ways of working, quality metrics, risk management
**Testing activities:** Review requirements, execute tests, analyze logs

## Testing Is Both Role and Act

- Tester (role) can do non-testing activities
- Developer (role) can and should do testing activities
- Test = confirming/falsifying a hypothesis
- Testing is **everywhere** — not just executing scenarios

## Testing = Challenging Assumptions

Testing is:
- Challenging assumptions and misconceptions
- Asking "why", "what if", "what will happen when"
- Test cases and scenarios
- Code reviews and static analysis
- Conversations at the coffee machine

## Key Quote

> "Testing is everywhere."
