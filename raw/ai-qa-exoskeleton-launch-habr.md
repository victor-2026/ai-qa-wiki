## Суть

Проверка обещания: "подключение тестирования на новый проект — 4 часа".

## 5 шагов подготовки

| Шаг | Чистое время | Реальное время |
|-----|--------------|---------------|
| Настройка проекта | 20 мин | 2-8 часов |
| Сборка Global Context | 30 мин | 30 мин |
| Подготовка доступов | 15 мин | 1-3 дня |
| Настройка инструментов | 30 мин | 2-4 часа |
| Договорённости с командой | 60 мин | 1-2 дня |

**Итого: 2.5 ч чистого vs 3-7 дней календарного**

## Ключевой вывод

> Бутылочное горлышко — не технология, а организация.

- Скилл `global-context-builder` за 30 минут анализирует 3 репозитория
- Получить доступ к этим репозиториям — может занять 3 дня

## Global Context Structure

```
testing/jira/Global-context/{PROJECT}/
├── services_endpoints.yaml     # P0: карта сервисов и URL
├── test_data.yaml              # P0: тестовые пользователи, креды
├── {PROJECT}_FULL_SPECIFICATION.md      # P1: бизнес-логика
├── {PROJECT}_TECHNICAL_REQUIREMENTS.md # P1: архитектура, стек
├── api_{service}.yaml                  # P1: OpenAPI спецификация
└── web_{service}.md                    # P2: роутинг, формы
```

## Приоритеты файлов

| Приоритет | Файлы | Значение |
|-----------|-------|----------|
| P0 | services_endpoints, test_data | Без них тесты не запустятся |
| P1 | Спецификации, OpenAPI | Качество сценариев |
| P2 | Описание фронтенда | Улучшает UI-тесты |
| P3 | actual_features.md | Генерируется автоматически |
