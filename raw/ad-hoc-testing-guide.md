# A Beginner's Guide to Ad Hoc Testing

## What is Ad Hoc Testing

Informal testing without predefined test scenarios. Random, unstructured approach — no documentation, test cases, or formal plans needed.

Most valuable **after formal testing is complete**.

## Types of Ad Hoc Testing

### 1. Monkey Testing
Random actions without planning. Three types:

| Type | Description | Best For |
|------|-------------|----------|
| **Brilliant Monkey** | Advanced users with complex needs | UX refinement |
| **Smart Monkey** | Regular users doing basic tasks | Common actions |
| **Dumb Monkey** | Totally random approach | Early development |

### 2. Buddy Testing
- 2 people: 1 developer + 1 tester
- Same module/application
- Quick bug identification
- Real-time code changes

### 3. Pair Testing
- 2 QA testers with different experience
- One tests, one documents
- Less experienced gains skills
- Focus on comprehensive testing

## Buddy vs Pair Testing

| Aspect | Buddy Testing | Pair Testing |
|--------|--------------|--------------|
| **Who** | Dev + QA | QA + QA |
| **Goal** | Quick bug resolution | Deep insights |
| **Bug fix speed** | Fast (dev on-site) | Slower |
| **Best for** | Resource-constrained teams | Broader perspective |

## Ad Hoc vs Exploratory Testing

| Aspect | Ad Hoc | Exploratory |
|--------|--------|------------|
| Structure | Unstructured | Structured, time-boxed |
| Goals | No defined outcomes | Defined outcomes |
| Plan | No plan | Clear testing plan |
| Documentation | Not required | Required |

## When to Use Ad Hoc Testing

**Use when:**
- Limited time and resources
- Experienced testers familiar with software
- Need to find unexpected bugs
- After regression testing
- UAT confirmation

**Don't use when:**
- No experience with software
- Bug found in formal test case
- Software is in beta

## Advantages

- **Flexible** — explore without constraints
- **Promotes creativity** — develop critical thinking
- **Simplifies testing** — saves time
- **Increases coverage** — complements formal methods
- **Identifies bugs** — throughout SDLC

## Disadvantages

- Cannot replace formal testing
- No guarantee of finding errors
- Not repeatable
